IVORY

IVORY is a CSS framework, specially created for web applications and committed to improve the experience of users. It gives you all the basic integrants to build your pages faster and easier.

About The Journey
  • The First Step

    The project IVORY was started in July 2012, as a side project by a Software Engineer named Lakshmikanth Vallampati. The initial idea of the project IVORY is to create the CSS library for forms.

  • v1

    The first version of IVORY Forms released.

  • v2

    The IVORY Forms enhanced by adding more components like grid-system(non-responsive), tooltips, tabs, accordion and toggle switch. IVORY Forms turned into IVORY Framework.

  • v3

    The responsiveness added to v2. IVORY source code published in GitHub for the first time.

  • 100,000 Downloads

    IVORY reached new milestone.

  • v4

    Satyasai Vallampati joined in IVORY core team. The IVORY Framework is revamped with new design system and advanced grid system. Version 4 released.

                  node_modules/ivoryui-ivory/ivory.min.css

Typography

Font Family

To maintain great readability on different screens and browsers IVORY uses Montserrat font as default font family. Montserrat font family has 10 differnt font-weights from 100 to 900. For good readability, IVORY prefers to use light(300), regular(400), medium(500), semi-bold(600), bold(700) font weights.

Note: If you want to use a different font family instead of suggested Montserrat font family, then don't import or use montserrat-font.css and use your own font css in your application.

Buttons

Buttons are user interface elements which allows users to take actions throughout the project. It is important that they have ample click space and help communicate the importance of their actions. IVORY provides 3 types of buttons.

Primary Buttons

IVORY primary buttons are defined with blue color. Use .btn-primary class along with the button style class.

Solid Buttons

Solid buttons are used to highlight the primary actions. To avoid confusing users, donâ€™t use more than one primary solid button within a section or view.

 
  
                    
                      <button class="iv-btn btn-primary">Confirm</button>

                      <button class="iv-btn btn-primary btn-sm">Confirm</button>

                      <button class="iv-btn btn-primary btn-ro">Confirm</button>

                      <button class="iv-btn btn-primary btn-ro btn-sm">Confirm</button>
Outline Buttons

Outline buttons are used to indicate the secondary actions, when they are along with the solid buttons. Multiple Outline buttons can be used in a section.

 
  
                    
                      <button class="iv-btn-o btn-primary">Confirm</button>

                      <button class="iv-btn-o btn-primary btn-sm">Confirm</button>

                      <button class="iv-btn-o btn-primary btn-ro">Confirm</button>

                      <button class="iv-btn-o btn-primary btn-ro btn-sm">Confirm</button>
Flat Buttons

Flat buttons have significant purpose, don't get confused with hyperlink. Flat buttons can be used inline scenarios where user can identify this as a button.

  
                    
                      <button class="iv-btn-f btn-primary">Confirm</button>

                      <button class="iv-btn-f btn-primary btn-sm">Confirm</button>

Tooltip

A tooltip provides a short details about a specific text or element on the UI in a pop-over. The tip is shown on mouse enter, and is hidden on mouse leave. The Tooltip doesn't support complex text or operations.

Tooltip Directions:

IVORY tooltip supports 8 directions, defined by using one of the following classes with the .tooltip-content class:
.top
.right
.bottom
.left
.top-left
.top-right
.bottom-right
.bottom-left

Tooltip Size:

The size of the tooltip content box can be changed. IVORY provides 3 different sizes, with the following classes:
.tooltip-xs
.tooltip-md
.tooltip-lg
